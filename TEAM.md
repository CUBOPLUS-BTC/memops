# Team and Authorship

MemOps is currently being built and maintained by a single developer for the CUBO+ evaluation stage.

## Current Roles Covered by the Same Person

- product direction
- technical implementation
- documentation
- repository maintenance
- demo preparation

## Why This File Exists

This note helps make the repository structure and authorship model explicit.

MemOps is intentionally scoped as a single-maintainer MVP. The project is documented in a way that allows future contributors to understand it, but it is not presented as if it already had a large team or formal governance structure.
