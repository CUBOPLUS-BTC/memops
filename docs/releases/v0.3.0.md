# MemOps v0.3.0  
  
## Highlights  
  
MemOps v0.3.0 establishes the current export baseline for the project.  
  
Included in this release:  
  
- configurable backend-backed transaction inspection  
- local raw transaction parsing  
- explicit RBF signaling analysis  
- first-pass why-stuck diagnosis  
- deterministic export of diagnosis artifacts:  
  - `<export_dir>/<txid>/analysis.json`  
  - `<export_dir>/<txid>/report.md`  
- surfaced artifact paths in text and JSON output  
- explicit CLI failure behavior when export cannot write artifacts  
- automated coverage for export success and failure paths  
- repository documentation, demo flow, and handoff material aligned with shipped behavior  
  
## Validation  
  
Release validation for v0.3.0 includes:  
  
- `uv run ruff format --check .`  
- `uv run ruff check .`  
- `uv run pytest -q`  
- manual CLI validation for:  
  - inspection mode  
  - JSON inspection mode  
  - why-stuck mode  
  - JSON why-stuck mode  
  - `--export-dir` artifact writing  
  - `MEMOPS_EXPORT_DIR` + `--export` artifact writing  
  
## Notes  
  
This release is intentionally analysis-first.  
  
Not included in v0.3.0:  
  
- wallet integration  
- automatic fee bumping  
- transaction signing  
- automatic rebroadcasting  
- CPFP execution  
- long-running daemon behavior  
  
## Reviewer notes  
  
There are no intended runtime behavior changes in this PR beyond aligning version metadata and packaging the release notes for the already-merged export baseline.
