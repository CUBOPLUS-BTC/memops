# Technical Documentation Index  
  
This directory contains project-level technical documentation for MemOps.  
  
The goal of this folder is to make the repository easier to review, easier to demo, and easier to continue as an open-source project after the CUBO+ evaluation stage.  
  
---  
  
## Files  
  
### `architecture.md`  
Explains the intended high-level structure of the project, including the trust model and the separation between adapters, domain logic, services, and CLI orchestration.  
  
### `assumptions.md`  
Documents what the MVP assumes, what it verifies locally, what it does not yet support, and where its current limits are.  
  
### `api-notes.md`  
Notes the backend endpoints and compatibility assumptions for mempool-compatible integrations.  
  
### `demo-script.md`  
Provides a step-by-step Demo Day flow, including what to show, what to say, and how to handle fallback scenarios.  
  
### `donation-readiness.md`  
Explains why the repository is structured as an open-source, reusable, and low-dependency project.  
  
### `evaluation-mapping.md`  
Maps repository artifacts to the CUBO+ evaluation criteria to make review easier.  
  
---  
  
## Documentation Philosophy  
  
MemOps is a small project, so the documentation should stay:  
  
- honest,  
- practical,  
- easy to review,  
- and tightly aligned with the actual state of the repository.  
  
The documentation should not pretend the project is larger or more mature than it really is.
