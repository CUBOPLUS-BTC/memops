# Strategy Folder Overview

This directory contains the non-technical and cross-functional documentation for MemOps.

These files serve two purposes:

1. to support the **CUBO+ evaluation requirements**, and
2. to give the project a clearer open-source narrative beyond the hackathon context.

Although only some of these files are strictly required by CUBO+, keeping the strategy documentation organized makes the repository easier to review and strengthens the overall professionalism of the project.

---

## Files in This Directory

### `problem.md`
Defines the real-world problem MemOps addresses: the gap between mempool visibility and operational response.

### `impact-analysis.md`
Explains the expected technical, operational, and ecosystem impact of the project.

### `operating-model.md`
Describes how MemOps is maintained as a small, public, single-maintainer open-source project.

### `business-model.md`
Frames MemOps as an open-source sustainability model rather than a forced SaaS startup.

### `roadmap.md`
Defines the MVP scope, priority levels, and what should be cut first if time becomes limited.

### `pitch-script.md`
Contains the non-technical narrative for Demo Day and related presentations.

### `worklog-link.md`
Links to the external Google Doc used to show progressive work and edit history over time.

---

## Notes for Reviewers

- MemOps is currently a **single-maintainer MVP**.
- The strategy documents are written in English to support future open-source use.
- The repository should be read as a project that is intentionally scoped, not as a fake large-scale company or mature commercial product.

---

## Relationship to the Technical Side

These documents should be read together with:

- `README.md`
- `src/README.md`
- `docs/architecture.md`
- `docs/assumptions.md`
- `docs/demo-script.md`

Together, they explain both **why** MemOps exists and **how** it is intended to work.
